1. การพิสูจน์ตัวตน (Authentication)
ฟังก์ชัน EmployeeSignin ใน Controller:

การรับข้อมูล: ใช้ Gin Framework เพื่อรับข้อมูลผู้ใช้จาก JSON ผ่าน POST request. ฟังก์ชันจะเช็คว่า JSON ที่ส่งมาถูกต้องหรือไม่ด้วย ShouldBindJSON.
การตรวจสอบฐานข้อมูล: ตรวจสอบว่าชื่อผู้ใช้ (username) ที่ผู้ใช้กรอกอยู่ในฐานข้อมูลหรือไม่ โดยใช้ GORM เพื่อดึงข้อมูลของผู้ใช้.
การตรวจสอบรหัสผ่าน: ใช้ฟังก์ชัน CheckPasswordHash เพื่อตรวจสอบว่ารหัสผ่านที่ผู้ใช้กรอกตรงกับแฮชที่เก็บไว้ในฐานข้อมูลหรือไม่.
2. การสร้างโทเค็น (Token Generation)
ฟังก์ชัน GenerateToken ใน Services:

สร้างโทเค็น JWT: ใช้ JwtWrapper เพื่อสร้างโทเค็น JWT โดยใช้ชื่อผู้ใช้ (username) เป็น Claims.
การเข้ารหัส: โทเค็นจะถูกเข้ารหัสด้วยคีย์ลับ (SecretKey) ที่ใช้ในการตรวจสอบความถูกต้องในอนาคต.
3. การจัดการโทเค็นใน Middleware
ฟังก์ชัน Authorizes ใน Middleware:

การตรวจสอบ Authorization Header: รับค่า Authorization header จาก request และตรวจสอบว่าเป็นรูปแบบที่ถูกต้อง (ต้องเริ่มต้นด้วย "Bearer ").
การตรวจสอบโทเค็น: ใช้ ValidateToken เพื่อตรวจสอบความถูกต้องของโทเค็น JWT ที่ส่งมาพร้อมกับ request. หากโทเค็นถูกต้อง จะอนุญาตให้ดำเนินการต่อไป.
การพิมพ์ข้อความ: ข้อความถูกพิมพ์ใน console เพื่อช่วยในการดีบักและติดตามการทำงาน.
4. การจัดการคีย์ลับ (SecretKey Management)
การตั้งค่า SecretKey:

การเก็บค่า SecretKey: ค่า SecretKey ควรจะถูกเก็บใน environment variable หรือในไฟล์คอนฟิก เพื่อความปลอดภัย (เช่น JWT_SECRET_KEY).
การเข้าถึง SecretKey: ฟังก์ชัน GetSecretKey จะช่วยดึงค่าคีย์ลับนี้ขึ้นมาเมื่อจำเป็น เช่น ใน JwtWrapper.
5. Frontend Service
การเชื่อมต่อ Frontend กับ Backend:

การใช้ Local Storage: เมื่อผู้ใช้เข้าสู่ระบบและได้รับโทเค็น JWT แล้ว โทเค็นจะถูกเก็บใน Local Storage ของเบราว์เซอร์. นี่ทำให้โทเค็นสามารถเข้าถึงได้สำหรับการเรียก API ที่ต้องการการยืนยันตัวตนในอนาคต.
การตั้งค่า Header สำหรับการเรียก API: เมื่อทำการเรียก API ที่ต้องการการยืนยันตัวตน เช่น /genders, จะมีการตั้งค่า Authorization header โดยส่งโทเค็นที่เก็บไว้ใน Local Storage.
6. การใช้ Token ใน Frontend
การส่ง Token: เมื่อทำการเรียก API ผู้ใช้จะต้องส่งโทเค็น JWT ที่ถูกเก็บไว้ใน Local Storage ใน header ของคำขอ.
การตรวจสอบ Token: Middleware ใน Backend จะทำการตรวจสอบว่าโทเค็นที่ส่งมานั้นถูกต้องและยังไม่หมดอายุ.
7. กระบวนการทำงานทั้งหมด
ผู้ใช้กรอกชื่อผู้ใช้และรหัสผ่านในหน้า Login ของ Frontend.
Frontend ส่งข้อมูลเข้าสู่ระบบไปยัง API ผ่าน POST request.
Backend ทำการตรวจสอบชื่อผู้ใช้และรหัสผ่าน.
หากข้อมูลถูกต้อง Backend จะสร้างโทเค็น JWT และส่งกลับไปยัง Frontend.
Frontend จะเก็บโทเค็นใน Local Storage.
ผู้ใช้สามารถเรียกใช้งาน API อื่น ๆ โดยส่งโทเค็นใน Authorization header.
Middleware ใน Backend จะตรวจสอบความถูกต้องของโทเค็นก่อนอนุญาตให้เข้าถึง API.
สรุป
การทำงานทั้งหมดนี้เป็นกระบวนการที่ช่วยให้การพิสูจน์ตัวตน (Authentication) เป็นไปอย่างปลอดภัยและมีประสิทธิภาพ โดยใช้ JWT เป็นเครื่องมือในการจัดการโทเค็น และการจัดการคีย์ลับอย่างเหมาะสมเพื่อป้องกันการเข้าถึงที่ไม่ได้รับอนุญาต.
